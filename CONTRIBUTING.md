Ways to contribute
- Open issues to report bugs or request features.
- Submit pull requests with bug fixes, improvements, or docs updates.
- Improve documentation, examples, or add tests.

Getting started
1. Fork the repository and create a branch for your change (use a descriptive branch name):

```bash
git checkout -b feat/my-feature
```

2. Make small, focused changes. Add tests where appropriate.

3. Commit with clear messages and push to your fork:

```bash
git add .
git commit -m "feat: short description of change"
git push origin feat/my-feature
```

4. Open a Pull Request against `main`. Describe what you changed and why. Link any related issues.

Guidelines
- Keep changes small and well-scoped.
- Follow the project's existing code style (TypeScript/React conventions for frontend; PEP8 for Python backend).
- Add or update tests when you modify behavior.
- Document public APIs and configuration changes in the README.

Branching & releases
- Work on feature branches and open PRs targeting `main`.
- The maintainer team will review PRs and merge after review and CI checks.

Code review
- Be responsive to review comments and keep PRs updated.

Thank you for helping build Nokast! If you have any questions, open an issue to start a conversation.
